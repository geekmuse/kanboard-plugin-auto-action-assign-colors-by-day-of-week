## Codebase Patterns
- **Timezone resolution helper**: Extract `DateTime` construction + `format('l')` into a shared `resolveDayOfWeek($ts)` private method whenever both `hasRequiredCondition()` and `getColorForDay()` need the day name. This avoids timezone drift and keeps the configurable timezone read in one place.
- **Timezone parameter list**: `DateTimeZone::listIdentifiers(DateTimeZone::ALL)` + `array_combine($ids, $ids)` provides the full IANA identifier list as a select dropdown. Prepend `'' => t('Server default')` as a sentinel; `resolveDayOfWeek()` falls back to `date_default_timezone_get()` when the param is empty.
- **Sentinel "No change" in color lists**: `array('' => t('No change')) + $this->colorModel->getList()` prepends an empty-string sentinel to every day's color dropdown. The existing `$color !== '' ` guard in `hasRequiredCondition()` already treats this as "skip" — no extra guard logic needed.
- **getParam API**: `$this->getParam($key)` returns the configured value for an action parameter — ActionManager pre-populates `$this->params` from `action_has_params` before `execute()` is called. Never query the DB to fetch action params.
- **Parameter key strategy (FIXED)**: `getActionRequiredParameters()` keys MUST be fixed English strings (e.g., `'Monday'`), never wrapped in `t()`. `DateTime::format('l')` always returns English day names. Using `t()` on keys stores translated names in `action_has_params` that can never be matched by the English lookup — permanent failure in non-English installs.
- **PSR-12 braces**: PHP classes and methods require opening brace on its own line (Allman style). Use `cytopia/phpcs:latest` Docker image to verify PSR-12 compliance when vendor/ is not yet bootstrapped.
- **docker-*.sh skip pattern**: Scripts in `scripts/` exit 0 with a SKIP message when vendor/ is missing. This allows pre-commit hooks to pass during early development before US-010 bootstraps composer/vendor.
- **PHP image split**: The `kanboard/kanboard` Docker image lacks the `tokenizer` extension (needed by PHPUnit 11 and PHPStan). Use `php:8.4-cli` (entrypoint `/bin/sh`) for test/phpstan/phpcs runners; use `kanboard/kanboard` (entrypoint `/bin/sh`) only for PHP lint (`php -l`) where tokenizer is not required.
- **Kanboard-agnostic test bootstrap**: PHPUnit tests use a `TestableAssignColorsByDayOfWeek` subclass that bypasses `Pimple\Container` entirely. Override `__construct(array $services)` (no parent call) and `__get(string $name)` to resolve DI services from a plain array. PHP guarantees class property defaults (e.g. `private $params = []`) are initialised before any constructor runs, so `setParam`/`getParam` work correctly on the subclass instance.
- **hasRequiredCondition() is the full guard**: All failure paths MUST be checked in `hasRequiredCondition()` — `doAction()` should be fully unconditional. This is the Kanboard action lifecycle contract enforced by ActionManager.
- **Day-of-week resolution**: Use `(new DateTime('now', new DateTimeZone($tz)))->setTimestamp($ts)->format('l')` to get the English day name from a Unix timestamp. Both `hasRequiredCondition()` and `getColorForDay()` use this pattern; US-005 will extract it into a shared private helper.
